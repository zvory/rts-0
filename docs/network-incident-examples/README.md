# Network Incident Examples

This directory preserves lag and networking incident evidence that can be read without private
agent memory. New incidents should use the capture package layout generated by
`scripts/capture-net-incident.mjs` so raw inputs, parser output, agent digest, artifact availability,
player notes, and neutral analysis stay together.

## Capture Commands

For the preserved Soupman/Alex regression fixture:

```bash
node scripts/capture-net-incident.mjs \
  --fixture soupman-alex \
  --out-dir /tmp/rts-soupman-alex-package \
  --force \
  --require-coverage command,snapshot,pathing,client-context
```

For a fresh beta incident with a known run id and UTC window:

```bash
node scripts/capture-net-incident.mjs \
  --beta \
  --from 2026-06-30T00:16:00Z \
  --to 2026-06-30T00:42:00Z \
  --run-id alex-s-lobby-1782778605186-000004 \
  --match-id 103 \
  --out-dir /tmp/rts-beta-incident \
  --require-coverage command,snapshot,pathing,client-context
```

The beta command shells through `scripts/fly-logs.sh`, which reads `FLY_API_TOKEN` from the
environment or ignored `.env` files. The package script records the query shape but never prints or
writes the token.

## Package Layout

A complete capture package contains:

- `README.md`: match/run identity, exact UTC window, build, participants, source files, and a neutral
  diagnosis paragraph.
- `raw/`: bounded run-id or UTC-window logs used by the parser.
- `parser/incident-summary.md`, `parser/incident-summary.json`, and `parser/incident-rows.tsv`:
  backwards-compatible parser outputs.
- `agent-digest.md` and `agent-digest.json`: standalone agent-readable digest.
- `key-metrics.json`: stable digest JSON for automated checks.
- `diagnostic-coverage.json`: command, snapshot, pathing, and client-context coverage gate output.
- `replay/replay.json` or `replay/UNAVAILABLE.md`.
- `db/db-summary.json` or `db/UNAVAILABLE.md`.
- `player-report-notes.md`: player-visible notes with absolute UTC timestamps.
- `analysis.md`: neutral supported, contradicted, unknown, and next diagnostic gap sections.
- `beta-evidence-checklist.md`: checklist for deployed build, run id, UTC window, focused query,
  parser command, source coverage, and unknowns.

## Beta Evidence Gate

Before starting any fixing plan from a fresh incident, confirm:

- The deployed build is recorded.
- The exact match run id is recorded.
- The UTC start and end are absolute timestamps.
- The log query is focused and bounded.
- The parser command is rerunnable from repo root.
- `diagnostic-coverage.json` has the expected command, snapshot, pathing, and client-context rows or
  makes missing coverage explicit.
- Replay and DB artifacts are copied, or their unavailable reason files explain what was missing.
- Player notes use absolute UTC timestamps.
- `analysis.md` lists supported, contradicted, unknown, and next diagnostic gaps without prescribing
  a fix.
